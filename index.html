MTH1W: Pseudocode Interpreter

by Mark Roxas. Last updated: Feb-08-2025

This tool allows you to run code inputted using the same format and commands (see Glossary of commands below) as written in the pseudocode of the Grade 9 Provincal Assessment of Mathematics for MTH1W. Please make sure javascript is enabled on your computer or mobile device. You will also find pseudocode samples from released assessment questions and sample tasks you can use to implement this tool. Glossary of Commands

        
set <variable> = <value>: Define a variable and assign it a value.

        
store user input as <variable>: Prompt the user to enter a value, then store it in the specified variable.

        
store input as <variable>: Prompt the user to enter a value, then store it in the specified variable.

        
<variable> = input <text>: Prompt the user to enter a value, then store it in the specified variable.Text must be enclosed in double quotes ("").

        
output <text>, <variable>: Print a combination of text and variables. Text must be enclosed in double quotes ("").

        
display <text>, <variable>: Print a combination of text and variables. Text must be enclosed in double quotes ("").

        
if <condition>: Start a conditional block. The block executes only if the condition evaluates to true.

        
else: Define an alternate block to execute if the preceding if condition is false.

        
end: Close an if or else block.

        
repeat loop <N> times: Execute the enclosed block exactly N times.

        
repeat loop until <condition>: Execute the enclosed block repeatedly until the condition becomes true.

        
end loop: Close a loop block.

        
Arithmetic Expressions: Use standard arithmetic operators (+, -, *, /, ^) in expressions to calculate values.

        
Exponentiation (^): Supports positive, negative, and decimal exponents. Example: result = 4 ^ 0.5 evaluates to 2.

   


        
Released Provincial Assessment Pseudocode

    Copy and paste the snippets of pseudocode into the interpreter and run the code. You can also modify the code to alter the output.
    

        

            Sample 1 (Sample Test 2024-2025)
        

        

            
                
set number = 0 
output "Enter a number."
store user input as number
stepOne = number ^ 2
stepTwo = 3 * stepOne - 4
output stepTwo
                

            		
            

        

            Sample 2 (Released Question 2023)
        

        

            
                
rateOfChange = -4
initialValue = 12
output "Enter the x-coordinate of the point."
store user input as xInput
output "Enter the y-coordinate of the point."
store user input as yInput
yValue = rateOfChange * xInput + initialValue
if yInput > yValue
output "The point is above the line"
else
output "The point is not above the line."
                

            		
               

                

            Sample 3 (Released Question 2023)
           

        

            
                
variableB = 0
variableC = 0
output "Enter the value of variable B."
store input as variableB
output "Enter the value of variable C."
store input as variableC
variableA = ((5 * variableC) - variableB)/4
output "The value of variable A is ", variableA, "."
                

            		
        

        

            Sample 4 (2024-2025 Sample Test)
           

        

            
                
amount = 0
interest = 0
output "Enter initial amount."
store user input as amount
output "Enter the simple interest rate as a percentage."
store user input as interest
finalAmount = amount + (amount * interest/100 * 5)
output "The final amount at the end of 5 years ", finalAmount, "."

 Sample 6 (Curriculum Sample Task#4 from E1.4)

        

            
                
set radius = 0
store user input as height
repeat loop 20 times
radius = radius + 1
volumeCylinder = 3.14 * (radius ^ 2) * height
volumeCone = (3.14 * (radius ^ 2) * height) / 3
output radius, " ", height, " ", volumeCylinder, " " , volumeCone
end loop
